## üìö Lessons Learned

### 1. **index.html and README.md Are Separate**
- Landing page (index.html) doesn't pull from README.md
- Must manually keep "Recent Features" and "Coming Soon" sections in sync
- Updated SKILL-Development-Ground-Rules.md to document this

### 2. **Visual Fixes Pattern: Test ‚Üí Report ‚Üí Fix**
- User noted: when in test/report/fix cycle for visual issues, just fix without asking
- Speeds up iteration for obvious adjustments

### 3. **User-Significant vs Developer-Significant Features**
- Logo/color changes are developer-significant (infrastructure)
- Wishlist/hide features are user-significant (behavior changes)
- "Recent Features" should highlight user-significant changes
