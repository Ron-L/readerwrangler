## What We Learned

### False Bug Investigation Waste
**Problem:** Spent development time investigating "Uncollected" collection persistence as a bug
**Root Cause:** Misunderstood that "Uncollected" is a hardcoded pseudo-collection by design, not a dynamic collection that should clear
**Impact:** Wasted time during testing, created false bug report in TODO.md
**Lesson:** Before reporting bugs, verify understanding of intended behavior vs actual behavior
**Prevention:** Document hardcoded UI elements clearly in code comments

### Recurring Version Confusion Pattern
**Problem:** Claude Code repeatedly confuses file versions (ORGANIZER_VERSION constant) with APP/project versions (README badge, git tags)
**Pattern:** During discussions, references wrong version type (e.g., "APP version 3.7.0" when meaning "file version v3.7.0")
**Impact:** Causes confusion during release prep, requires repeated clarification
**Example:** During this release, confused readerwrangler.js v3.7.0 with APP version v3.7.0 (which doesn't exist - APP was at v3.7.2)
**Mitigation:** Always use explicit prefixes ("file version" vs "APP version") in discussions
**Documentation:** Added to CHANGELOG.md Development Notes for future reference

### Documentation Duplication Burden
**Problem:** Maintaining both LOG.md and CHANGELOG.md was redundant work
**Discovery:** LOG.md had duplicate release entries plus unique @RULES and development insights
**Analysis:** Compared industry standards - most open-source projects use CHANGELOG.md only
**Decision:** Eliminate LOG.md, follow "Keep a Changelog" standard
**Benefit:** Less confusion, no duplication, reduced maintenance burden

### Version Numbering Slip
**Problem:** Changed readerwrangler.js from v3.7.0 to v3.7.0.a without incrementing fix portion
**Correct Process:** Should have gone v3.7.0 → v3.7.1.a (for functional change) → v3.7.1 (after testing)
**What Happened:** Went v3.7.0 → v3.7.0.a → v3.7.1 (skipped proper increment)
**Impact:** Minor - corrected before release, but shows process not followed perfectly
**Reminder:** Even small functional changes (variable rename) deserve version increment
